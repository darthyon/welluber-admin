## 0. Goals

1. **Consolidate** all mock data into `lib/mock-data/` with factories + seed — single source of truth.
2. **Rename** Wallet → Account throughout (types, features, components, imports).
3. **Cross-link** all entity IDs so data forms a coherent graph.
4. **Fill** each entity to 10 instances.
5. **Extract** all inline mock data from components/pages.
6. **Add ephemeral mutations** — create/update/delete works during session, resets on refresh.
7. **Zero breakage** — after every phase: `pnpm typecheck && pnpm build && pnpm lint:design`.

### Phase 3 — Add Ephemeral Mutation Hooks (additive, medium risk)

Create domain hooks that wrap mock data with create/update/delete.

**Step 3.1: Create hooks/data-hooks.ts**

```bash
hooks/data-hooks.ts
```

Each hook pattern:

```ts
"use client"
import { useState, useCallback } from "react"

// Seed imports from lib/mock-data
import { MOCK_ORGANIZATIONS } from "@/lib/mock-data"

export function useOrganizations() {
  const [items, setItems] = useState(MOCK_ORGANIZATIONS)
  const add = useCallback((item: Organization) => setItems(prev => [item, ...prev]), [])
  const update = useCallback((id: string, patch: Partial<Organization>) =>
    setItems(prev => prev.map(i => i.id === id ? { ...i, ...patch } : i)), [])
  const remove = useCallback((id: string) =>
    setItems(prev => prev.filter(i => i.id !== id)), [])
  return { data: items, isLoading: false, error: null, add, update, remove }
}
```

Repeat for: `useBrands`, `useServiceProviders`, `useEmployees`, `useMembers`, `useAdmins`, `usePolicies`, `useClaims`, `useAccounts`, `useAuditLogs`, `useVouchers`.

**Step 3.2: Update consuming pages**

Replace direct imports with hook calls:

```ts
// Before:

// After:
import { useOrganizations } from "@/hooks/data-hooks"
const { data: orgs } = useOrganizations()
const filtered = useMemo(() => orgs.filter(...), [orgs])
```

**This is the biggest change.** ~15 pages/components affected. Do it one page at a time.

## 5. Risk Flags

1. **Phase 1 is the riskiest single change.** Touching ~15 files across multiple layers. Mitigation: do it in small commits, verify after each file group.

2. **Employee type reconciliation.** `Employee` (types.ts) vs `EmployeeDirectoryItem` (inline) have different shapes. If they don't reconcile cleanly, the employee directory table may break. Mitigation: extend `Employee` with the extra fields needed by the table, don't force them down.

3. **Policy types.** `PolicyListItem` extends `BenefitPolicy` with `groupCount`, `orgName`, `assignedOrgNames`. These are presentational fields derived from cross-references. The factory should compute them, not hardcode them.

4. **Dashboard data.** `topOrgs` and `topSPs` are deliberately left inline. They represent aggregated/sorted data, not entities. If you want them linked to real org/SP data, they should be derived from the seed rather than hardcoded.

5. **AdvancedFilterSheet** uses `walletModel` in `AdvancedFilters` interface. Renaming this field requires updating the filter sheet component AND every page that passes advanced filters. Check: `organizations/page.tsx`, `accounts/page.tsx`, and any other consumer.

6. **No faker dependency.** Factories will use hand-written arrays of names/values + index-based variation. If you later add `@faker-js/faker`, the factory functions can be updated without changing anything else.
