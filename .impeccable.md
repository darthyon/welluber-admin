## Design Context

### Users
HR administrators and benefits managers at corporate wellness platforms. They configure complex benefit policies (eligibility rules, pool strategies, service groups) for employee wellness programs. Context: desktop-first admin console, used during working hours, often multi-tasking between policy configuration and employee management.

### Brand Personality
Precision Infrastructure — inspired by Linear and Vercel. Clean, information-dense, quietly premium. Every pixel intentional. No decoration without information purpose.

Voice: Confident, direct, technical but approachable. Three words: **precise**, **structured**, **trustworthy**.

Emotional goals: Users should feel in control and confident that the system handles complexity reliably.

### Aesthetic Direction
- **Theme**: Light mode default, full dark mode support
- **Typography**: Geist Variable, tight tracking on headings, strict semibold (600) max weight
- **Color**: Deep indigo primary accent (`oklch(0.457 0.24 277.023)`), warm stone borders, OKLCH-based palette
- **Spacing**: Tight within, generous between. 4px base unit. Use gap for sibling spacing.
- **Surfaces**: Border-defined, not shadow-defined. `rounded-lg` for cards, `rounded-4xl` for triggers.
- **Density**: Compact but breathable. Information-dense without feeling cramped.

### Design Principles
1. **Whitespace is structural** — generous between sections, tight within groups.
2. **Borders define surfaces** — no heavy shadows, depth through border contrast.
3. **Reserve primary color for CTAs** — everything else is neutral.
4. **Left-aligned asymmetry** — avoid centering everything; asymmetric layouts feel more designed.
5. **Consistent spacing scale** — use the 4pt system (4, 8, 12, 16, 24, 32, 48), never arbitrary values.
